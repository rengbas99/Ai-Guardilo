# Chrome Web Store Resubmission Checklist

Follow these steps to ensure your extension meets all store policies and passes the review process.

## 1. Description & Metadata
- [ ] **Remove Keyword Stuffing:** Ensure the description does not contain comma-separated lists of brand names (e.g., "ChatGPT, Claude, Gemini..."). Use the template in `STORE_DESCRIPTION.md`.
- [ ] **Descriptive Title:** Ensure the title is clear and matches the functionality.
- [ ] **Single Purpose:** The description should clearly state one primary purpose (e.g., "Privacy-first PII redaction for web applications").

## 2. Privacy & Data Safety
- [ ] **Privacy Policy:** Ensure the Privacy Policy URL in the developer dashboard points to a valid, accessible page (like your `docs/privacy.html` or the raw file on GitHub).
- [ ] **Data Usage Disclosure:** In the "Data usage" tab of the dashboard, you MUST declare that you handle PII. 
    - Select: "Personal communications", "Personal identification information", and "Financial and payment information".
    - Explanation: "The extension scans user-input text locally to detect sensitive PII (names, emails, bank details) and provide redaction tools before the data is sent to AI platforms."
- [ ] **Local Processing:** Emphasize that all data is processed locally. Ensure "No data is sent to external servers" is checked if applicable.

## 3. Permissions (Principle of Least Privilege)
- [ ] **Justification:** Be prepared to provide a one-sentence justification for each permission:
    - `storage`: "Needed to store anonymized risk stats and user settings locally."
    - `clipboardWrite`: "Needed to allow users to copy redacted text for manual pasting in sensitive environments."
- [ ] **Match Patterns:** Ensure match patterns are as specific as possible. You have already limited these to major AI and productivity sites.

## 4. Branding & Visuals
- [ ] **Icon Designs:** Ensure your icons do not look identical to official Google/Chrome/OpenAI icons.
- [ ] **Screenshots:** Provide clear, high-quality screenshots showing the extension *in action* (e.g., showing the underlines and the "Update" popup). Avoid using misleading mockups.

## 5. Resubmission Steps
1. Update the **Description** field with the content from `STORE_DESCRIPTION.md`.
2. Double-check the **Privacy** and **Data Usage** sections.
3. Upload the new `.zip` package if any code changes were made (though currently, the fixes are documentation-based).
4. In the "Review instructions" box, add a short note:
   > "This update addresses the 'Spam and Placement' violation (Yellow Argon). We have removed the keyword-heavy brand list from the description and replaced it with a more descriptive, policy-compliant functional overview. No changes to permissions or data collection (still 100% local)."
